## Règle tactile tablette - boutons dans le Design

- Pour les boutons créés dynamiquement dans un post-it du Design Jeedom, ne pas se limiter à addEventListener('click').
- Sur tablette, le clic synthétique peut être intercepté par la couche tactile du Design Jeedom.
- La méthode robuste validée est : attribut/propriété directe onclick, ontouchend, onpointerup, avec return false.
- Ajouter aussi un verrou court anti double-déclenchement.
- Cette règle s’applique notamment aux boutons Classic / Paper / Tape dans les options du post-it.
- Marqueur de référence : POSTITDESIGN_VISUAL_STYLE_TABLET_INLINE_TOUCH_V2.
- Ne pas revenir à une solution seulement basée sur addEventListener('click') pour les boutons dynamiques du Design.

## Règle validée - bouton + Post-it avec menu

- Le clic principal sur `+ Post-it` doit toujours créer un post-it immédiatement.
- Ne pas transformer le clic principal en ouverture de menu.
- Ne pas utiliser clic droit, double-clic, appui long ou tap spécial : non fiable dans les Designs Jeedom et sur tablette.
- Ne pas remplacer complètement le rendu natif Jeedom de la commande `create_postit`, sinon le déplacement du bouton dans le Design peut casser.
- La solution validée est une petite pastille visible sur le bouton.
- Clic sur la pastille = ouverture du menu.
- Le menu doit rester cantonné au Design.
- Ne pas utiliser `position:fixed` pour le menu : cela le positionne par rapport à la fenêtre du navigateur et le fait sortir du Design.
- Le menu validé est en `position:absolute`, borné dans le conteneur du bouton.
- Le masquage des post-it doit être non destructif : ne pas supprimer les lignes `plan`, ne pas déplacer les équipements, conserver les positions.
- Remote GitHub obligatoire en SSH : `git@github.com:Gruikgroin67/post-it-design.git`.
- Marqueur classe : POSTITDESIGN_CMD_MENU_INLINE_DOT_DESIGN_BOUND_V6
- Marqueur AJAX : POSTITDESIGN_CMD_MENU_INLINE_DOT_AJAX_V5

## v1.0.39 - 2026-05-05 - Couleur rapide tactile

Fonction ajoutee :
- Pastilles de couleur rapide directement dans les options du post-it.
- Couleurs sauvegardees via l'action AJAX setColorFromDesign.
- Couleur appliquee immediatement au DOM avant meme la reponse AJAX.
- Correction du style Tape : la bande haute jaune fixe est remplacee par une bande neutre/translucide compatible avec toutes les couleurs.

Regle tactile importante :
- Pour les boutons utilisables dans le Design Jeedom sur tablette, ne pas se limiter a click ou touchend.
- Jeedom peut intercepter touchend/click sur tablette iOS ou Android.
- Pour les boutons critiques, utiliser aussi touchstart et pointerdown avec capture:true, passive:false si possible.
- Ajouter un verrou anti double-execution, par exemple lockUntil = Date.now() + 650/750 ms.
- Garder click/pointerup/touchend pour PC et compatibilite, mais declencher l'action tactile principale des touchstart/pointerdown quand l'action est simple et sans saisie clavier.
- Ne pas mettre de stopPropagation global en capture sur le conteneur parent d'une rangee de boutons : cela peut empecher les boutons enfants de recevoir l'evenement.
- Le parent doit seulement stopper la propagation en phase bubble si besoin, pour eviter que Jeedom recupere l'evenement apres le bouton.

Marqueurs associes :
- POSTITDESIGN_QUICK_COLOR_OPTIONS_V1
- POSTITDESIGN_QUICK_COLOR_OPTIONS_AJAX_V1
- POSTITDESIGN_QUICK_COLOR_TOUCH_APPLY_FIX_V1
- POSTITDESIGN_QUICK_COLOR_RECTANGLE_FIX_V1
- POSTITDESIGN_QUICK_COLOR_TOP_BAND_FIX_V1
- POSTITDESIGN_TAPE_TOP_BAND_NEUTRAL_V1
- POSTITDESIGN_QUICK_COLOR_TOUCHSTART_FIX_V1

A ne pas refaire :
- Ne pas appliquer la couleur sur title/message/footer, car cela cree des rectangles parasites.
- Ne pas garder une bordure tape jaune fixe du type rgba(245,230,140,.70).
- Ne pas utiliser position:fixed pour les menus ou panneaux du Design.

## Priorite post-it - contour orange/rouge

Fonction :
- postit_priority = normal | important | urgent.
- Normal : pas de contour supplementaire.
- Important : contour orange via box-shadow, pas via border.
- Urgent : contour rouge via box-shadow, pas via border.
- Ne pas utiliser border pour la priorite : cela peut changer la taille du post-it et perturber le Design Jeedom.
- Attribution depuis les options du post-it directement dans le Design.
- Sauvegarde via AJAX setPriorityFromDesign.
- Tactile : utiliser touchstart/pointerdown + click/touchend/pointerup avec verrou anti double-execution, comme pour couleur rapide.

Marqueurs :
- POSTITDESIGN_PRIORITY_OUTLINE_RENDER_V1
- POSTITDESIGN_PRIORITY_OUTLINE_OPTIONS_V1
