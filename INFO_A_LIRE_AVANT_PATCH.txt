## Règle tactile tablette - boutons dans le Design

- Pour les boutons créés dynamiquement dans un post-it du Design Jeedom, ne pas se limiter à addEventListener('click').
- Sur tablette, le clic synthétique peut être intercepté par la couche tactile du Design Jeedom.
- La méthode robuste validée est : attribut/propriété directe onclick, ontouchend, onpointerup, avec return false.
- Ajouter aussi un verrou court anti double-déclenchement.
- Cette règle s’applique notamment aux boutons Classic / Paper / Tape dans les options du post-it.
- Marqueur de référence : POSTITDESIGN_VISUAL_STYLE_TABLET_INLINE_TOUCH_V2.
- Ne pas revenir à une solution seulement basée sur addEventListener('click') pour les boutons dynamiques du Design.

## Règle validée - bouton + Post-it avec menu

- Le clic principal sur `+ Post-it` doit toujours créer un post-it immédiatement.
- Ne pas transformer le clic principal en ouverture de menu.
- Ne pas utiliser clic droit, double-clic, appui long ou tap spécial : non fiable dans les Designs Jeedom et sur tablette.
- Ne pas remplacer complètement le rendu natif Jeedom de la commande `create_postit`, sinon le déplacement du bouton dans le Design peut casser.
- La solution validée est une petite pastille visible sur le bouton.
- Clic sur la pastille = ouverture du menu.
- Le menu doit rester cantonné au Design.
- Ne pas utiliser `position:fixed` pour le menu : cela le positionne par rapport à la fenêtre du navigateur et le fait sortir du Design.
- Le menu validé est en `position:absolute`, borné dans le conteneur du bouton.
- Le masquage des post-it doit être non destructif : ne pas supprimer les lignes `plan`, ne pas déplacer les équipements, conserver les positions.
- Remote GitHub obligatoire en SSH : `git@github.com:Gruikgroin67/post-it-design.git`.
- Marqueur classe : POSTITDESIGN_CMD_MENU_INLINE_DOT_DESIGN_BOUND_V6
- Marqueur AJAX : POSTITDESIGN_CMD_MENU_INLINE_DOT_AJAX_V5
